useState is used for managing local state within a functional component. It allows you to declare state variables and provides a way to update them, triggering re-renders of the component when the state changes. useEffect, on the other hand, is used for handling side effects in your components. Side effects are operations that interact with the outside world or perform actions that aren't directly related to rendering, such as data fetching, subscriptions, timers, or manual DOM manipulations.

By default, useEffect runs after every render of the component (both initial render and subsequent updates). The optional dependency array (the second argument to useEffect) controls when the effect will re-run.
